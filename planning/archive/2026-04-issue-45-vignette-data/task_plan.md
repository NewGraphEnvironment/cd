# Task Plan — Pre-compute peace-fwcp vignette data (#45)

## Goal

Stop the peace-fwcp vignette from re-fetching ~144 COG range requests on every CI render. Pre-compute the heavy data (zonal extracts and a tiny pre-cropped departure raster), ship as bundled assets, vignette renders cheaply.

## Steps

- [ ] **`data-raw/peace_fwcp_vignette_data.R`** — generates `inst/vignette-data/peace_fwcp.rds` with: regional `ts`, regional `bl_early`, regional `ano`, regional `trn`, regional `cmp`, regional `cmp_pct`; per-ecoregion list of `ts`/`ano`/`trn`/`cmp_pct`. Sets GDAL retry env vars at top so the regen itself is robust.
- [ ] **`inst/vignette-data/peace_fwcp_departure_tmean.tif`** — also generated by the script: `cd_crop(tmean_annual.tif, aoi)` then mean(2015:2025) − mean(1951:1980) saved as a single-band GeoTIFF, masked to AOI. Final raster ~64×39 cells, well under 50 KB.
- [ ] **`vignettes/peace-fwcp.Rmd`** — replace live `cd_extract` / `cd_compare` / `cd_crop` calls with `readRDS()` + `terra::rast()` against the bundled assets. Keep the catalog table chunk live (small JSON, useful demo).
- [ ] **Note** at top of vignette referencing `data-raw/` provenance.
- [ ] **Code-check** the diff before commit (no leftover live network calls except the one demo-friendly catalog read).
- [ ] **Render local** to confirm timing is ~seconds, output unchanged in content.
- [ ] **PR** with SRED reference in body.

## Out of scope

- KOTL vignette (small enough, works fine)
- Removing the catalog read entirely — that's the demonstration of `cd_catalog()` for readers
- `cd_compare()` p-value (#43, separate)

## Validation

- [ ] Local render time drops from ~6 min to <30 s
- [ ] Vignette content is byte-equivalent (or near-equivalent — figures will be visually identical)
- [ ] CI passes on PR
