# Findings

## Convention reference

`r-packages` conventions in this repo's CLAUDE.md (under "Vignettes")
say explicitly:

> Vignettes that need external resources (DB, API, STAC): Do NOT use
> the `.Rmd.orig` pre-knit pattern — it breaks bookdown figure
> numbering. Instead, separate data generation from presentation.

Recommended pattern (from the convention):

1. `data-raw/vignette_data.R` runs the queries, saves results as
   `.rds` to `inst/testdata/` (or `inst/vignette-data/`)
2. Vignette loads `.rds` files, all chunks run live during pkgdown
   build (but they're now computationally cheap)
3. Note at top of vignette: "Data generated by `data-raw/script.R`"
4. bookdown controls all chunks — figure numbers, cross-refs work

This issue is exactly that.

## CI failure mode reproduced

Post-merge run for #44 failed with:

```
[crop] file does not exist: https://stac-era5-land.s3.us-west-2.amazonaws.com/prcp_annual.tif
```

— a transient `/vsicurl/` flake. Same code, same URL, passed on the
immediately-following Release v0.1.2 run. Confirms the failure mode is
network/intermittent rather than a real broken reference.

## Object size estimates

- Regional `ts` tibble: ~24 variable-period rows × 76 years = ~1,800 rows.
  Negligible.
- Per-ecoregion `ts` × 5: same shape, ~9,000 rows total. Few hundred KB.
- Pre-cropped tmean departure raster: 64 × 39 single-band float32
  (cropped to FWCP AOI bbox) ≈ 10 KB compressed.
- Total bundled `inst/vignette-data/`: well under 1 MB.

## Decisions

- **Keep `cd_catalog()` live** in the vignette. It's a small JSON read
  and demonstrates the consumer entry point for readers — value of
  showing it working > saving milliseconds.
- **Drop `cd_extract` / `cd_baseline` / `cd_anomaly` / `cd_trend` /
  `cd_compare` from the vignette body**, replace with `readRDS()` of
  pre-computed results. The narrative still references those functions
  (we say "we ran cd_extract") but the chunks load results.
- **Drop `cd_crop` for the spatial-pattern map**, replace with
  `terra::rast(system.file(...))` of the pre-cropped raster.
